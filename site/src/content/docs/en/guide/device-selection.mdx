`HikCamera::devices()` returns a `Devices` collection backed by
`MV_CC_EnumDevices` over `GigE + USB` transports. From there you can pick the
device you actually want to open.

## Enumerate

```rust
let devices = hik.devices()?;
for device in devices.iter() {
    println!("{:?}", device.info());
}
```

Each `Device` holds:

- the borrowed `'hik` lifetime (it cannot outlive `HikCamera`)
- a copy of the raw `MV_CC_DEVICE_INFO`
- a parsed `DeviceInfo` with normalized fields across transports

See [Device info fields](/reference/device-info-fields/) for the full field
reference.

## Pick one

`Devices` provides several selector helpers that all **consume** the
collection and return a single `Device`:

| Selector | When to use | Failure mode |
| --- | --- | --- |
| `default()` | First available device | `NoDevice` if list is empty |
| `serial_number(value)` | Stable across reboots, recommended | `DeviceNotFound` / `MultipleDevices` |
| `user_name(value)` | User-configured readable name (e.g. `"left-cam"`) | `DeviceNotFound` / `MultipleDevices` |
| `ip(value)` | GigE device by current IP | `DeviceNotFound` / `MultipleDevices` |
| `mac(value)` | GigE or MAC-tagged device; case-insensitive | `DeviceNotFound` / `MultipleDevices` |

<Aside type="tip" title="Prefer serial_number for production">
  Serial numbers are physically engraved and never change. IPs can be
  re-assigned by DHCP, and user names can be wiped by a factory reset. Pick
  `serial_number` whenever the camera is in a fixed installation.
</Aside>

## `Device::open()`

`device.open()` returns a `Camera`. Under the hood it:

1. Calls `MV_CC_CreateHandle` to allocate a per-device handle.
2. Calls `MV_CC_OpenDevice` with exclusive access
   (`MV_ACCESS_Exclusive`).
3. If either step fails, calls `MV_CC_DestroyHandle` to release the partially
   created handle before returning the error.

You can check whether `open()` is likely to succeed without actually opening
the device with `device.is_accessible()`, which wraps
`MV_CC_IsDeviceAccessible` — useful when iterating several devices to find
the first free one.

## Transports

The current enumerator always covers `MV_GIGE_DEVICE | MV_USB_DEVICE`.
`DeviceInfo` parses fields for other transports (CameraLink, 1394, GenTL
variants) too, so future enumerators can reuse it without changes.
