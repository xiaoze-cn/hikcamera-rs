`Stream` exposes two writer builders — `save_image` for a single still and
`save_video` for a multi-frame recording. Both hand you a writer object you
feed `Frame`s into; the writer owns the SDK save/record lifecycle for that
file.

## Save a single image

```rust
let mut image = stream.save_image("capture.bmp")?;
image.write_frame(&frame)?;
image.finish()?;
```

`save_image(path)` uses sensible defaults. Use `save_image_with` to override
the format, quality, or both:

```rust
use hikcamera::{ImageFormat, SaveOptions};

let opts = SaveOptions {
    format: ImageFormat::Jpeg,
    quality: 90,                        // JPEG quality, 1..=100
};
let mut image = stream.save_image_with("capture.jpg", opts)?;
image.write_frame(&frame)?;
image.finish()?;
```

### Supported image formats

| `ImageFormat` | Notes |
| --- | --- |
| `Bmp` | Default when the path ends in `.bmp`. Lossless, large. |
| `Jpeg` | Lossy. Set `quality` 1..=100. |
| `Png` | Lossless with compression. |
| `Tiff` | Lossless. Multi-page if you call `write_frame` multiple times. |
| `Raw` | Vendor-specific raw container. |

The path extension is a hint; the actual format is whatever you pass in
`SaveOptions.format`. If you omit the format, the wrapper infers it from the
extension.

## Record an AVI video

```rust
use std::time::{Duration, Instant};

let mut video = stream.save_video("clip.avi", 30.0)?;     // 30 fps
let started = Instant::now();
while started.elapsed() < Duration::from_secs(10) {
    let frame = stream.take_frame(Duration::from_secs(1))?;
    video.write_frame(&frame)?;
}
let video = video.finish()?;
println!("wrote {} frames over {:?}", video.frame_count, video.elapsed);
```

### `VideoWriter` lifecycle

<Steps>

1. **Construct** with `stream.save_video(path, fps)` or
   `stream.save_video_with(VideoOptions { … })`.
2. **First `write_frame`** lazily calls `MV_CC_StartRecord` with the frame's
   pixel format and dimensions — the first frame establishes the format.
3. **Each `write_frame`** calls `MV_CC_InputOneFrameEx` and bumps
   `frame_count`.
4. **`finish()`** calls `MV_CC_StopRecord` and returns a `Video` struct with
   `frame_count`, `frame_rate`, `width`, `height`, `pixel_type`, `elapsed`.

</Steps>

### Recording rules

<Aside type="danger" title="One recording per Stream at a time">
  The SDK supports a single active recording per device handle. Constructing
  a second `VideoWriter` while the first is still open returns
  `HikCameraError::RecordingInProgress`. Always `finish()` before starting
  the next.
</Aside>

<Aside type="note" title="Frames must match the source format">
  The wrapper does not convert frames on its way into the recorder. If you
  want to record a different pixel format than the sensor produces, call
  `stream.convert_frame(&frame, format)?` first and feed the converted frame
  to the writer.
</Aside>

## High-level shortcuts

If you don't want to manage the writer objects directly, `Camera` exposes two
convenience methods that wrap the full capture-and-save flow:

```rust
// One-liner: open stream → take frame → save → return Camera.
let frame = camera.take_image("snapshot.bmp")?;

// One-liner: open stream → record for N seconds → return Camera.
let camera = camera.take_video("clip.avi", Duration::from_secs(5))?;
```

These are great for prototyping; for production loops you'll usually want the
explicit `save_image_with` / `save_video_with` forms so you can control
options and timing.

## Full example

The bundled example at `crates/hikcamera/examples/camera.rs` does both in
sequence:

<Tabs>
  <TabItem label="capture image">

    ```rust
    let mut stream = camera.stream()?;
    let frame = stream.take_frame(Duration::from_secs(1))?;
    let mut image = stream.save_image(Path::new(IMAGE_PATH))?;
    image.write_frame(&frame)?;
    image.finish()?;
    ```

  </TabItem>
  <TabItem label="record video">

    ```rust
    let mut stream = camera.stream()?;
    let mut video = stream.save_video(Path::new(VIDEO_PATH), 30.0)?;
    let started = Instant::now();
    while started.elapsed() < Duration::from_secs(10) {
        let frame = stream.take_frame(Duration::from_secs(1))?;
        video.write_frame(&frame)?;
    }
    let _video = video.finish()?;
    ```

  </TabItem>
</Tabs>

Build and run with `just build-examples` then
`cargo run --package hikcamera --example camera` (requires a real camera).
