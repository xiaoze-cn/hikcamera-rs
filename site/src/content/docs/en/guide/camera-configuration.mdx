`Camera` owns the parameter node map of the open device. The wrapper exposes
typed helpers for the common nodes, plus a generic `get_node` / `set_node`
pair that works on any GenICam node.

## Typed helpers

| Domain | Getters / setters | Underlying nodes |
| --- | --- | --- |
| Exposure | `get_exposure`, `set_exposure(f32)` | `ExposureTime` (Float) |
| Exposure auto | `get_exposure_auto`, `set_exposure_auto(AutoMode)` | `ExposureAuto` (Enum) |
| Gain | `get_gain`, `set_gain(f32)` | `Gain` (Float) |
| Gain auto | `get_gain_auto`, `set_gain_auto(AutoMode)` | `GainAuto` (Enum) |
| ROI | `get_roi`, `set_roi(Roi)` | `Width`, `Height`, `OffsetX`, `OffsetY` (Int) |
| Trigger | `get_trigger`, `set_trigger(Trigger)` | `TriggerMode`, `TriggerSource`, `TriggerActivation` |
| Frame rate | `get_fps`, `set_fps(Fps)` | `AcquisitionFrameRateEnable`, `AcquisitionFrameRate` |
| Gamma | `set_gamma(f32)` | direct `MV_CC_SetGammaValue` (pixel-format aware) |
| Bayer conversion | `set_bayer_conversion(BayerOptions)` | `MV_CC_SetBayerCvtQuality`, `MV_CC_SetBayerFilterEnable` |
| Bayer CCM | `set_bayer_ccm(CcmOptions)` | `MV_CC_SetBayerCCMParamEx` |
| Stream options | `set_stream(StreamOptions)` | `MV_CC_SetImageNodeNum`, `MV_CC_SetGrabStrategy`, `MV_CC_SetOutputQueueSize` |

```rust
let mut camera = device.open()?;

camera.set_exposure(8000.0)?;             // us
camera.set_gain(3.0)?;                    // dB
camera.set_roi(Roi { width: 1920, height: 1080, offset_x: 0, offset_y: 0 })?;
camera.set_trigger(Trigger::Off)?;        // free-running
camera.set_fps(Fps::Target(30.0))?;
```

## Generic node access

For any node the helpers do not cover, use `get_node` / `set_node` /
`execute_node`:

```rust
use hikcamera::{NodeValue, NodeInput, NodeType};

match camera.get_node("BalanceWhiteAuto")? {
    NodeValue::Enum(value) => println!("current: {:?}", value.current),
    NodeValue::Float(value) => println!("current: {}", value.current),
    _ => {}
}

camera.set_node("BalanceWhiteAuto", "Once")?;
camera.execute_node("BalanceWhiteAuto")?;   // command node

// Inspect the node type if you don't know it ahead of time.
match camera.node_type("UserSetSelector")? {
    NodeType::Enum => { /* ... */ },
    other => return Err(Error::unsupported_node("UserSetSelector", other.name()).into()),
}
```

`NodeValue` and `NodeInput` are the read-side and write-side sum types. The
helpers above are essentially typed shortcuts that call `get_node` / `set_node`
under the hood.

<Aside type="caution" title="Type mismatches are typed errors">
  Reading a Float node as Enum (or vice versa) returns
  `HikCameraError::NodeValueMismatch`. Writing a string to an Integer node
  returns `HikCameraError::NodeInputMismatch`. You never get a silent
  conversion.
</Aside>

## Trigger modes

`Trigger` is an enum covering the common combinations:

| Variant | Behavior |
| --- | --- |
| `Off` | Free-running, frame rate driven by `AcquisitionFrameRate` |
| `Software` | Frames wait for `camera.execute_node("TriggerSoftware")` |
| `Hardware{ source, activation }` | Frames wait for an electrical trigger on the configured line |

```rust
use hikcamera::Trigger;

camera.set_trigger(Trigger::Software)?;
// ... later, when you want the next frame:
camera.execute_node("TriggerSoftware")?;
// Then stream.take_frame(timeout) will return that triggered frame.
```

## Frame rate

`Fps` distinguishes between a **target** rate and **device-limit** mode:

```rust
use hikcamera::Fps;

camera.set_fps(Fps::Target(30.0))?;     // request 30 fps
camera.set_fps(Fps::DeviceLimit)?;      // let the sensor run at its max
let current = camera.get_fps()?;
```

`get_fps` reads both `AcquisitionFrameRateEnable` and the resulting rate so you
can tell whether the limit is your target or the device ceiling.
