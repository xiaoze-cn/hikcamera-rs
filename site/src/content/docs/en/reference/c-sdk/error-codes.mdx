`MvErrorDefine.h` defines the status codes returned by most
`MV_CC_*` / `MV_GIGE_*` / `MV_USB_*` / `MV_CAML_*` functions. Success is
`MV_OK` (`0x00000000`); failures are `0x80000000`-style literals that read
as negative `i32` values.

The Rust wrapper's `Status` newtype, `HikCameraError::Sdk { status }`, and
the `SDK_ERRORS` lookup table are all derived from this header. See
[Error model](/developer/error-model/).

## USB, upgrade, image processing, render, liquid lens

<Aside type="note" title="Smaller families">
  The remaining code families (USB errors, upgrade errors, image processing
  errors, render errors, liquid lens errors) are smaller and live entirely in
  the source file. Browse them at:
  [`crates/hikcamera-sys/include/MvErrorDefine.h`](https://github.com/xiaoze-cn/hikcamera-rs/blob/main/crates/hikcamera-sys/include/MvErrorDefine.h).
  ISP algorithm codes are in `MvISPErrorDefine.h` — see
  [ISP error codes](/reference/c-sdk/isp-error-codes/).
</Aside>
