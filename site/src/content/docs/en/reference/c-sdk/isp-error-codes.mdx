`MvISPErrorDefine.h` defines the `MV_ALG_*` family of status codes returned
by ISP / image-processing paths in the SDK (`MV_CC_ISPProcess`,
`MV_CC_ConvertPixelTypeEx` for some conversions, etc.). They use the same
`0x80000000`-style encoding as the main error codes but a different numeric
range, so they appear as a distinct group in the lookup table.

## Denoise / purple-fringe correction

<Aside type="note" title="Algorithm-specific codes">
  The denoise and purple-fringe-correction (PFC) sub-families
  (`MV_ALG_E_DENOISE_*`, `MV_ALG_E_PFC_*`) live in the source header and
  are typically only surfaced when calling those specific algorithm paths.
  Browse them at:
  [`crates/hikcamera-sys/include/MvISPErrorDefine.h`](https://github.com/xiaoze-cn/hikcamera-rs/blob/main/crates/hikcamera-sys/include/MvISPErrorDefine.h).
</Aside>
