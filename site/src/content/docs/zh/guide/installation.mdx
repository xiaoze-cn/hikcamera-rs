`hikcamera-rs` 只支持 Windows。构建需要 `libclang`（用于 `bindgen`）和 MSVC
工具链（用于链接厂商导入库），两者都通过 `pixi` 自动准备，日常流程基本只有一条命令。

## 前置要求

<Aside type="note" title="打开仓库前你需要什么">
  - Windows 10 / 11（x64 或 x86）
  - 系统已安装海康 MVS 运行时，**或** 仓库里 `crates/hikcamera-sys/lib/` 下随仓库发布的
    DLL（已自带）
  - PATH 上有 [pixi](https://pixi.sh)
  - PATH 上有 [just](https://github.com/casey/just)
</Aside>

## 1. 克隆并初始化

<Steps>

1. 克隆仓库：

   ```sh
   git clone https://github.com/xiaoze-cn/hikcamera-rs
   cd hikcamera-rs
   ```

2. 跑一次 setup。它会安装 pixi 环境、在只有版本化 `libclang.dll` 时复制一份
   Windows 命名版本，并注册 `lefthook` pre-commit 钩子：

   ```sh
   just setup
   ```

3. 验证 wrapper 层能干净构建：

   ```sh
   just build
   ```

</Steps>

## 2. 始终通过 `just` 调用 cargo

每条 `just` recipe 都会设置 `LIBCLANG_PATH`，并把 pixi 环境前置到
`PATH`。直接跑 `cargo` 会因为环境没配置好而失败，报 `Unable to find libclang`。

<Tabs>
  <TabItem label="常用 recipe">
    | Recipe | 作用 |
    | --- | --- |
    | `check` | `cargo check --workspace` |
    | `build` | `cargo build --workspace` |
    | `build-examples` | `cargo build --workspace --examples` |
    | `test` | `cargo test --workspace` |
    | `fmt-check` | 跑 `rustfmt` 检查（提交前跑） |
    | `pre-commit` | `fmt-check` + `check`（lefthook 在 commit 时跑的） |
    | `clean` | `cargo clean` |
  </TabItem>
  <TabItem label="临时给 cargo 传参">
    recipe 名后的额外参数会被原样转发给 cargo：

    ```sh
    just test --lib error     # cargo test --workspace --lib error
    ```

    没有对应 recipe 的命令，可以前缀 `just` 的 shell 环境跑，或者从 `justfile`
    头部复制环境变量自己设。
  </TabItem>
</Tabs>

## 4. 把 wrapper 加进你的 crate

<Tabs>
  <TabItem label="Cargo.toml（git）">
    ```toml
    [dependencies]
    hikcamera = { git = "https://github.com/xiaoze-cn/hikcamera-rs.git" }
    ```
  </TabItem>
  <TabItem label="仅 Windows target">
    该 crate 构建时链接 `MvCameraControl.lib`，运行时加载
    `MvCameraControl.dll`。两者都已随仓库放在
    `crates/hikcamera-sys/lib/{win32,win64}/` 下，所以使用者**不**需要
    系统级安装 MVS SDK 就能编译——只有真的连相机跑时才需要。

    ```toml
    [target.'cfg(windows)'.dependencies]
    hikcamera = { git = "https://github.com/xiaoze-cn/hikcamera-rs.git" }
    ```
  </TabItem>
</Tabs>

<Aside type="caution" title="bindgen 重构很慢">
  `bindgen` 会解析完整的 MSVC + 厂商头文件集合。除非任务就是改 FFI，否则不要动
  `crates/hikcamera-sys/wrapper.h`、`include/*.h` 或 `build.rs`——每次改动都
  触发很长的重构。
</Aside>
