`HikCamera::devices()` 返回 `Devices` 集合，底层是跨 `GigE + USB` 传输层的
`MV_CC_EnumDevices`。接下来从集合里挑出真正要打开的那一台。

## 枚举

```rust
let devices = hik.devices()?;
for device in devices.iter() {
    println!("{:?}", device.info());
}
```

每个 `Device` 持有：

- 借用的 `'hik` 生命周期（不能比 `HikCamera` 活得长）
- 原始 `MV_CC_DEVICE_INFO` 的一份拷贝
- 跨传输层归一化后的 `DeviceInfo`

完整字段看 [设备信息字段](/zh/reference/device-info-fields/)。

## 选一台

`Devices` 提供几个选择器，全部**按值消费** `Devices` 并返回单个 `Device`：

| 选择器 | 何时用 | 失败模式 |
| --- | --- | --- |
| `default()` | 取第一台 | 空列表返回 `NoDevice` |
| `serial_number(value)` | 跨重启稳定，**推荐** | `DeviceNotFound` / `MultipleDevices` |
| `user_name(value)` | 用户给的可读名（如 `"left-cam"`） | `DeviceNotFound` / `MultipleDevices` |
| `ip(value)` | 按 GigE 当前 IP 选 | `DeviceNotFound` / `MultipleDevices` |
| `mac(value)` | 按 MAC 选，大小写不敏感 | `DeviceNotFound` / `MultipleDevices` |

<Aside type="tip" title="生产环境优先 serial_number">
  序列号是物理刻印的，不会变。IP 可能被 DHCP 重派，用户名可能被恢复出厂清空。
  固定安装的场景里尽量用 `serial_number`。
</Aside>

## `Device::open()`

`device.open()` 返回 `Camera`，内部依次：

1. 调用 `MV_CC_CreateHandle` 分配设备级句柄。
2. 用独占访问（`MV_ACCESS_Exclusive`）调 `MV_CC_OpenDevice`。
3. 任何一步失败时先调 `MV_CC_DestroyHandle` 释放半创建的句柄，再返回错误。

想在真正 open 前判断能否打开，可以用 `device.is_accessible()`，它包装
`MV_CC_IsDeviceAccessible`——遍历多台设备找第一台空闲时很有用。

## 传输层

当前枚举器总是覆盖 `MV_GIGE_DEVICE | MV_USB_DEVICE`。`DeviceInfo` 同时解析了
其它传输层（CameraLink、1394、GenTL 各变体）的字段，将来新增专用枚举器可以
直接复用，无需改动。
